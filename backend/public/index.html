Documentación de la API SpendTracker

Descripción

Esta es una API RESTful para la gestion de gastos.

Endpoints disponibles

1. Registrar Usuario

Método: POST

Ruta: /auth/register

Descripción: Permite registrar un nuevo usuario.


POST http://localhost:3000/auth/register

Headers:
Content-Type: application/json

Body (JSON):
{
    "email": "emailexample",
    "password": "passwordexample"
}

2. Iniciar Sesión

Método: POST

Ruta: /auth/login

Descripción: Inicia sesión y devuelve un token JWT.


POST http://localhost:3000/auth/login

Headers:
Content-Type: application/json

Body (JSON):
{
    "email": "emailexample",
    "password": "passwordexample"
}

3. Crear un gasto

Método: POST

Ruta: spend

Descripción: Crea un nuevo gasto.


POST http://localhost:3000/spend

Headers:
Authorization: Bearer {token}
Content-Type: application/json

Body (JSON):
{
"title": "Pago de luz",
"amount": 1500,
"description": "Factura mensual de electricidad",
"category": "Servicios",
"email": "usuario@example.com",
"sharedEmail": "compartido@example.com"
}

4. Obtener Todos los gastos

Método: GET

Ruta: /spend

Descripción: Obtiene una lista de todos los gastos


GET http://localhost:3000/spend

Headers:
Authorization: Bearer {token}

5. Eliminar un gasto

Método: DELETE

Ruta: /spend/:id

Descripción: Elimina un gasto por su ID.


DELETE http://localhost:3000/spend/{id}

Headers:
Authorization: Bearer {token}

ENDPOINTS DE BOOKTAGS

No estan en uso actualmente

6. Filtrar Libros

Método: GET

Ruta: /libros/filter

Descripción: Filtra los libros por (title, yearPublished, genre), devuelve los libros que coincidan con estos parametros


GET http://localhost:3000/libros/filter?param=value

Headers:
Authorization: Bearer {token}

7. Obtener Libro por ID

Método: GET

Ruta: /libros/:id

Descripción: Obtiene un libro por su ID.


GET http://localhost:3000/libros/:id

Headers:
Authorization: Bearer {token}

9. Actualizar un Libro

Método: PUT

Ruta: /libros/:id

Descripción: Actualiza la información de un libro existente.


PUT http://localhost:3000/libros/:id

Headers:
Authorization: Bearer {token}
Content-Type: application/json

Body (JSON):
{
    "title": "Título Actualizado",
    "author": "Autor Actualizado"
}

11. Obtener libro por title

Método: GET

Ruta: /libros/title/:tile

Descripción:Obtener un libro por su titulo


GET http://localhost:3000/libros/title/:title

Headers:
Authorization: Bearer {token}

12. Obtener Libros Ordenados

Método: GET

Ruta: /libros/sort

Descripción: Obtiene una lista de libros ordenados por el campo especificado en los parámetros de consulta.


GET http://localhost:3000/libros/sort?sort={campo}&order={asc|desc}

Headers:
Authorization: Bearer {token}

Parámetros de Consulta:
- sort: Campo por el cual ordenar (por defecto: title). Ejemplo: title, yearPublished, genre, pages.
- order: Dirección del orden (por defecto: asc). Valores posibles: asc o desc.
